# 数学作业讲解视频 - Gemini 互动思考记录

> 来源：Gemini 3.1 Pro 对话 (2026-02-20)
> 链接：https://gemini.google.com/share/e197950e8702

## 一、初始构想

拍照上传一道题（含图形和文字，如几何题），生成讲解视频，视频中有图形动画、高亮、详细步骤及涉及的定理。

## 二、核心假设质疑

### "生成视频"并非最佳实践

- 延迟高：服务端渲染几何动画（如 Manim）需几十秒到几分钟
- 交互性差：视频是线性的，用户无法控制图形缩放、旋转或单步调试
- 业界实践：头部竞品（Photomath、Gauthmath、微软数学）采用"结构化数据驱动的前端动态渲染"

### "拍照直接生成精确几何动画"技术壁垒

- 多模态模型处理手绘几何图时存在"空间幻觉"
- 合理路径：图像转形式化数据 → 符号计算引擎验证 → 结构化动画脚本 → 客户端/服务端渲染

## 三、系统架构（三大核心层）

### 感知层：多模态信息解构

- 图像预处理：透视裁剪、二值化去噪、分离文本与图形区域
- 文本/公式提取：转换为 LaTeX 格式
- 几何图形逆向工程：转化为形式化语言（GeoGebra XML / TikZ / JSON）
- 动态约束求解：根据已知几何约束重算坐标，确保动画几何精确

### 认知层：逻辑推理与剧本生成

- 神经符号系统验证：符号计算系统（CAS）或几何定理证明器二次校验
- 剧本生成：详细推导步骤 + 定理提取 + 时间轴与动画状态映射

### 表现层：动态演示生成

- **路径 A（推荐）**：前端原生动态渲染（JSXGraph / GeoGebra Web），后端仅返回 JSON + TTS 数据
- **路径 B（备选）**：服务端 Manim 硬渲染 MP4 视频

### 外部能力与 API

| 类别 | 推荐方案 | 用途 |
|------|----------|------|
| 多模态大模型 | GPT-4o / Gemini / Claude | 图文联合理解，提取 LaTeX 与几何形式化描述 |
| 符号计算引擎 | SymPy / GeoGebra API / Lean 4 | 校验推导逻辑，重算图形坐标 |
| 动画渲染 | Manim（服务端）/ JSXGraph（客户端） | 几何动画生成 |
| 语音合成 | Azure AI Speech / ElevenLabs | 讲解语音 + 音素级时间戳对齐 |

## 四、成本测算

### 商业化场景（路线 A：客户端渲染）

单次约 ¥0.20，可支撑"每日免费 3 题 + 包月 15 元"的商业模式。

### 商业化场景（路线 B：服务端视频）

单次 ¥0.30-0.40，晚高峰并发渲染成本是业务死穴。

## 五、场景重定义 → 极致方案

**关键转折**：仅供两个孩子使用（初三 + 高一），不计成本追求极致效果。

### 不妥协技术栈

- **视觉引擎**：Manim（3Blue1Brown 开发的开源数学动画引擎）
- **代码生成**：Claude Sonnet 4.6（最新，SWE-bench 79.6%，100 万 token 上下文）
- **环境管理**：uv 管理 Python 虚拟环境
- **语音**：ElevenLabs（顶级情感表现）
- **部署**：本地设备，渲染成本归零

### 系统管线（Agentic Workflow）

```
阶段 1：多模态数学解构（Vision to Math）
  └── 上传照片 → Claude 输出结构化 JSON（LaTeX + 条件 + 解题步骤）

阶段 2：剧本与代码编译（Math to Code）
  └── 解题步骤 + Manim System Prompt → Claude 生成 Python 动画代码
  └── 约束：禁止硬编码坐标，必须用几何约束计算

阶段 3：沙盒执行与自愈循环
  └── 执行 Manim CLI → 报错则自动送回 Claude 修复（最多 3 次重试）
  └── 成功率从 40% 提升到 95%+

阶段 4：音视频合成
  └── TTS 生成语音 → FFmpeg 混音 → 输出 1080P/60FPS MP4
```

### 针对初高中痛点

- **初三动点问题**：Manim 的 ValueTracker + always_redraw，可视化点运动时面积/比例的连续变化
- **高一立体几何**：ThreeDScene 摄像机旋转，将立体几何降维为平面几何视角

## 六、三级交叉验证架构（确保 0% 偏差）

核心原则：大模型无法数学自证，必须引入神经符号系统。

### 第一级：代数系统前置求解（CAS Pre-computation）

- Claude 仅提取条件，生成 SymPy 求解脚本
- 本地执行 SymPy 获取绝对数学真值
- 将真值作为不可篡改常量注入后续 Prompt

### 第二级：几何到代数的降维断言

- 强制建立隐藏坐标系
- 在 Manim 代码中插入 `assert` 语句进行运行期校验
- 断言失败 → 渲染崩溃 → 截获异常 → 送回 Claude 纠错
- 示例：验证直线垂直于平面 → `assert abs(np.dot(vec_v, vec_n)) < 1e-7`

### 第三级：形式化定理证明器（可选，竞赛级）

- 引入 Lean 4 形式化证明语言
- 代码通过编译 = 证明 100% 成立

### 完整验证工作流

```
[Vision] 传入题目图片
  → [Extract] 提取约束，生成 SymPy 脚本
  → [Compute] 本地执行 SymPy，获取数学真值
  → [Prompting] 真值作为硬约束 → 生成解题步骤 + Manim 代码（含 assert）
  → [Render & Validate] 执行 Manim → assert 失败则熔断重试（最多 3 次）→ 成功输出视频
```
