# 测试策略：图片上传预览 + OCR 结果展示优化

## 1. 测试范围

### 在范围内
- **缩略图展示**：图片消息气泡中显示缩略图替代"[图片]"文字
- **OCR 结果展开/收起**：展开按钮交互、上图下文布局、过渡动画
- **图片放大查看**：模态框/lightbox 弹出、关闭（点击遮罩/ESC）
- **加载状态反馈**：本地预览零延迟、OCR 加载动画、超时/失败提示
- **异常处理**：图片加载失败占位图、OCR 失败提示文字、无图片时禁用放大

### 不在范围内
- 后端 OCR 服务逻辑（已有测试覆盖）
- 后端图片增强处理
- 图片持久化存储（R2）
- OCR 文本编辑功能

## 2. 测试类型与策略

### 2.1 单元测试（优先级：高）
- **目标**：验证新增/修改的 React 组件逻辑
- **工具**：Jest + React Testing Library
- **覆盖重点**：
  - 缩略图组件渲染（有图片 / 无图片 / 加载失败）
  - 展开/收起状态切换逻辑
  - 模态框打开/关闭逻辑
  - OCR 文本展示（有文本 / 空文本 / 加载中）
  - 图片 URL 生成与清理（Object URL 生命周期）

### 2.2 组件集成测试（优先级：高）
- **目标**：验证组件间数据流和交互
- **工具**：React Testing Library
- **覆盖重点**：
  - 图片上传后消息气泡正确显示缩略图
  - 点击展开按钮后 OCR 文本正确渲染
  - 消息列表中多条图片消息独立管理展开状态
  - 加载状态到完成状态的过渡

### 2.3 端到端测试（优先级：中）
- **目标**：验证完整用户流程
- **方式**：手动测试（项目当前无 E2E 框架）
- **覆盖重点**：
  - 上传图片 → 看到缩略图 → 展开 OCR → 放大查看 → 关闭
  - 页面刷新后图片显示占位图（Object URL 丢失）
  - 移动端浏览器兼容性

### 2.4 视觉/样式测试（优先级：中）
- **方式**：手动验证
- **覆盖重点**：
  - 缩略图尺寸（200-240px 宽度）和比例
  - 展开/收起过渡动画流畅性
  - 模态框遮罩层样式
  - 暗色模式兼容（如适用）
  - 响应式布局（手机/平板/桌面）

## 3. 测试优先级矩阵

| 场景 | 优先级 | 测试类型 | 风险等级 |
|------|--------|----------|----------|
| 缩略图替代"[图片]"文字 | P0 | 单元+集成 | 高 |
| OCR 文本展开/收起 | P0 | 单元+集成 | 高 |
| 图片加载失败占位图 | P0 | 单元 | 中 |
| OCR 失败提示 | P0 | 单元 | 中 |
| 模态框放大查看 | P1 | 单元+集成 | 中 |
| ESC 键关闭模态框 | P1 | 单元 | 低 |
| 加载动画展示 | P1 | 单元 | 低 |
| 页面刷新后占位图 | P2 | 手动E2E | 低 |
| 展开/收起动画 | P2 | 手动视觉 | 低 |

## 4. 测试环境

- **单元/集成测试**：Jest + jsdom（Node 环境，模拟 DOM）
- **手动测试**：本地开发服务器（`npm run dev`），Chrome + 移动端模拟
- **测试数据**：准备 fixture 图片文件（小图/大图/损坏图片），mock OCR API 响应

## 5. 质量门禁

- 所有 P0 测试用例通过
- 新增代码无 TypeScript 类型错误（`npm run build` 通过）
- 无 CRITICAL/HIGH 级别代码审查问题
- 现有测试套件不被破坏（`uv run pytest` 全部通过）

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Object URL 内存泄漏 | 浏览器内存增长 | 组件卸载时 `revokeObjectURL`，单元测试验证 |
| 大图片导致气泡撑开 | UI 错乱 | CSS `max-width` + `object-fit`，视觉测试 |
| 前端无 E2E 测试框架 | 交互流程依赖手动验证 | 加强组件级集成测试覆盖 |
| 项目前端测试基础设施不明 | 可能需要配置 Jest | 调研阶段确认，必要时补充配置 |
