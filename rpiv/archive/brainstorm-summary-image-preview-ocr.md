# 需求摘要：图片上传预览 + OCR 结果展示优化

## 产品愿景
- **核心问题**：用户上传题目图片后，聊天框只显示"[图片]"文字，看不到自己上传了什么，也看不到 OCR 中间结果，体验割裂
- **价值主张**：让图片上传和 OCR 识别过程对用户透明可见，提升信任感和使用体验
- **目标用户**：AIXue 的学生用户（初高中），主要通过手机拍照上传作业题目
- **产品形态**：AIXue Web 前端解题页面的 UI 优化

## 核心场景（按优先级排序）
1. **缩略图展示**：用户上传图片后，聊天气泡中显示缩略图替代"[图片]"文字
2. **OCR 结果展开**：用户点击缩略图下方展开按钮，查看压缩后的图片和 OCR 识别出的文本
3. **图片放大查看**：用户点击缩略图，弹出模态框查看完整大图
4. **加载状态反馈**：上传期间缩略图立即显示，OCR 文本区域显示加载动画

## 产品边界
- **MVP 范围内**：
  - 缩略图替代"[图片]"文字
  - 展开/收起按钮 + OCR 文本只读展示
  - 点击缩略图放大查看（模态框/lightbox）
  - 上传中加载动画
  - 展开区域上图下文布局
  - OCR 失败时展开区域显示提示文字
- **明确不做**：
  - 后端图片增强（旋转校正、对比度优化）
  - OCR 文本可编辑/修正
  - 图片持久化存储（R2）— 单独功能
- **后续版本考虑**：
  - 图片持久化（R2 方案，已有 todo）
  - OCR 文本可编辑并重新提交解题
  - 后端图片增强处理

## 已知约束
- 压缩后图片仅存在前端内存（Object URL），刷新页面后丢失，显示占位图
- 后端 API 已返回 `question` 字段（OCR 文本），无需新增后端接口
- 展开区域默认收起状态

## 各场景功能要点

### 场景1：缩略图展示
- **功能点**：用户发送图片消息后，气泡中显示图片缩略图（而非"[图片]"文字）
- **关键交互**：缩略图尺寸适中（建议宽度 200-240px），保持图片比例
- **异常处理**：图片加载失败时显示默认占位图 + "图片加载失败"文字

### 场景2：OCR 结果展开
- **功能点**：缩略图下方有一个小展开按钮（如"查看识别结果 ▼"），点击展开显示：上方为压缩后的图片（较大尺寸），下方为 OCR 识别的文本
- **关键交互**：默认收起，点击切换展开/收起，展开区域有过渡动画
- **异常处理**：OCR 失败或空文本 → 展开区域显示"未能识别图片中的文字"提示

### 场景3：图片放大查看
- **功能点**：点击缩略图弹出模态框/lightbox，显示完整大图
- **关键交互**：点击遮罩层或关闭按钮关闭模态框，支持 ESC 键关闭
- **异常处理**：无图片数据时禁用点击放大

### 场景4：加载状态反馈
- **功能点**：图片发送后立即在气泡中显示缩略图（本地预览），OCR 文本区域显示加载动画
- **关键交互**：缩略图从本地 File 对象生成（零延迟），加载动画在 AI 回复返回后替换为 OCR 文本
- **异常处理**：请求超时或失败时，加载动画替换为错误提示
