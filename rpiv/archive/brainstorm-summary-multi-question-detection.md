## 需求摘要

### 产品愿景
- **核心问题**：用户上传的题目图片常包含多道题，当前系统无法区分处理，不完整题目浪费资源，多题混合解答质量差
- **价值主张**：智能识别图片中的多道题目，自动过滤不完整题目，让用户按需选择解答
- **目标用户**：初中/高中学生，使用手机拍照上传试卷或作业题目
- **产品形态**：在现有 AIXue Web 端解题流程中增强，对话式交互

### 核心场景（按优先级排序）
1. **单题快速通道**：图片中只有 1 道完整题 → 跳过选题直接解答
2. **多题选择**：图片中有多道完整题 → 按钮组让用户选择解哪题或全部解答
3. **不完整题过滤**：部分题目不完整 → 标记提示，不进入解题流程
4. **全部不完整**：所有题目都不完整 → 提示用户重新拍照
5. **user_hint 快捷路径**：用户附带文字说明"第14题" → 跳过多题检测，直接聚焦处理

### 产品边界
- **MVP 范围内**：
  - 多题检测（数量、编号、完整性判断）
  - 对话中按钮组选题 UI
  - 单次最多 5 题限制
  - 同一会话逐题解答 + 可追问
  - user_hint 兼容（有 hint 跳过检测）
- **明确不做**：
  - 图片裁剪/校正（后续单独做）
  - 用户手动框选题目区域
  - 选题超时自动处理
  - 不完整题目内容展示
- **后续版本考虑**：
  - 题目区域裁剪 + 校正后的图片输出
  - 多张图片合并处理

### 已知约束
- 多题检测需要额外一次 LLM 调用（增加延迟和成本）
- 单次最多处理 5 道题，超出提示用户分批拍照

### 各场景功能要点

#### 场景1：单题快速通道
- 功能点：检测到 1 道完整题 → 自动进入解题，无需用户确认
- 关键交互：用户无感知，体验与当前一致
- 异常处理：无

#### 场景2：多题选择
- 功能点：检测到 2-5 道完整题 → AI 回复"检测到N道题目" + 按钮组
- 关键交互：按钮组包含每道题的题号 + "全部解答"按钮；不完整题灰显 + "不完整"标签
- 异常处理：超过 5 题 → 只展示前 5 道，提示"检测到N道题目，最多同时处理5道，建议分批拍照"

#### 场景3：全部解答（逐题）
- 功能点：用户点击"全部解答" → 按顺序逐题调用解题流程
- 关键交互：每题解答有清晰分隔标记（"**第X题**"标题），同一会话中展示
- 异常处理：某题解答失败 → 跳过该题继续下一题，最后汇总提示

#### 场景4：全部不完整
- 功能点：没有任何完整题目 → AI 回复"未检测到完整题目，请重新拍照确保题目完整"
- 关键交互：提示信息 + 用户可重新上传
- 异常处理：无

#### 场景5：user_hint 快捷路径
- 功能点：有文字说明时 → 沿用现有 RECOGNITION_FOCUSED_PROMPT，跳过多题检测
- 关键交互：与当前流程一致
- 异常处理：无
