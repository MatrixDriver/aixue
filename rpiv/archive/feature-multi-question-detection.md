# 多题图片智能识别与选题

## 动机与背景

用户上传的题目图片经常包含多道题目（如试卷照片），当前系统会尝试识别所有内容但一次性处理，导致：
- 不完整的题目（如只拍到一半的第13题）也被送入解题流程，浪费资源且结果无意义
- 用户无法指定只解其中一道题
- 多题混在一起处理，解答质量下降

## 期望行为

1. **题目检测阶段**：OCR 识别后，先分析图片中包含哪些题目，判断每道题的完整性
2. **不完整题目过滤**：自动检测不完整的题目（如只有部分内容可见），标记为"不完整"并告知用户，不进入解题流程
3. **用户选择**：当检测到多道完整题目时，提示用户选择：
   - 选择其中一道题解答
   - 选择全部解答（逐题处理）
4. **逐题处理**：如果用户选择多题，按顺序一题一题处理，每题独立生成解答，而非一次性混合处理

## 用户场景

1. 学生拍了一页试卷，里面有第13、14题，其中13题只拍到一半。系统提示"检测到2道题目：第13题（不完整，无法处理）、第14题（完整）"，自动选定第14题进行解答
2. 学生拍了完整的两道题，系统询问"检测到第5、6题，请选择要解答的题目"，学生选择"全部解答"，系统依次解答每道题
3. 学生拍照时附带文字说明"第14题"，系统直接聚焦第14题，跳过选择步骤（现有的 user_hint 机制）

## MVP 定义

- 后端新增"题目检测"步骤：OCR 后用 LLM 分析题目数量、编号、完整性
- 后端返回检测结果给前端（题目列表 + 完整性状态）
- 前端展示题目选择 UI（单选/全选）
- 不完整题目灰显 + 提示文字
- 选择后逐题调用解题流程
- 用户附带文字说明时（user_hint），跳过选择直接处理

## 备选方案

- 方案A：检测步骤在后端完成，前端展示选择 UI（推荐，交互更好）
- 方案B：前端让用户手动框选题目区域（实现复杂，暂不考虑）
