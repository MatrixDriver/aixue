# AIXue 管理后台

## 动机与背景

当前模型配置、成本查看等运营操作依赖 .env 环境变量和手动查日志，缺乏可视化管理界面。随着功能增多（模型拆分、成本优化等），需要一个统一的 Web 管理后台支撑日常运营。

## 期望行为

1. Web 管理界面，运营人员可通过浏览器访问
2. 支持模型配置动态切换（OCR 模型、推理模型等），无需重启服务
3. 成本统计面板：按时间、学科、模型维度查看 API 调用量和成本
4. 用户管理：查看用户列表、使用情况等基本信息

## 用户场景

1. 运营发现某模型性价比更好，在管理后台直接切换，立即生效
2. 运营查看本周 API 成本趋势，判断优化效果
3. 运营查看活跃用户及其使用频次

## MVP 定义

1. 模型配置页面：展示当前模型配置，支持在线修改
2. 成本统计页面：基础的调用量和 token 消耗统计
3. 认证保护：管理后台需要独立的管理员认证

## 备选方案

- 独立前端项目 vs 集成到现有 Next.js 前端的 /admin 路由
- 配置存储：数据库 vs Redis vs 配置文件
