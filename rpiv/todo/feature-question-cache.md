# 题目识别结果缓存机制

## 动机与背景

题目存在大量重复（尤其教材习题、真题）。对识别出的题目文本做哈希，命中缓存直接返回解答，可省掉重复的 LLM 调用。在实际运营中可节省相当比例的调用量。

## 期望行为

1. OCR 识别出题目文本后，计算文本哈希
2. 查询缓存：如果命中，直接返回已有的解答结果
3. 未命中时正常调用 LLM 解题，并将结果写入缓存
4. 缓存支持失效/更新机制（如模型升级后可清除旧缓存）

## 用户场景

1. 多个学生上传同一道教材习题，只有第一个需要调用 LLM，后续直接命中缓存
2. 同一学生重复上传同一道题，秒级返回

## MVP 定义

1. 题目文本 hash 计算（归一化处理：去空格/标点统一后 hash）
2. 缓存存储（Redis 或数据库）：key=题目hash，value=解答结果
3. 命中/未命中的统计指标
4. 缓存过期策略（如 30 天 TTL）

## 备选方案

- 精确匹配（hash）vs 模糊匹配（语义相似度）
- 缓存层：Redis vs 数据库 vs 文件系统
- 是否支持相似题目的模糊命中（需要向量检索）
